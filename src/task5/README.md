Данный скрипт - класс QuestionN5, унаследованный от QuestionBase.
Он используется для автоматической генерации различных вариантов Задачи 5 и набора тестов для неё.
**Условие задачи:**
Студенту необходимо написать программу на C, которая:
- Считывает число N — количество строк в тексте
- Построчно считывает N строк (каждая строка содержит одно предложение длиной не более 100 символов)
- Находит строку с заданным критерием (вариативно):
    - Наибольшей или наименьшей длиной (в символах)
    - Наибольшим или наименьшим количеством слов
- Выводит найденную строку и значение критерия в формате: <Значение>: <строка>
Запрещено использование библиотечных функций обработки строк (<string.h>)

**Инициализация:**
При запуске класс получает seed для воспроизводимости
Случайным образом определяется критерий поиска из 4 вариантов:
- Наибольшая длина строки
- Наименьшая длина строки
- Наибольшее количество слов
- Наименьшее количество слов
Гарантируется, что во входных данных нет строк с одинаковым значением критерия

**Основные атрибуты:**
- criteria_list - список словарей с конфигурациями критериев (тип метрики, режим min/max, текстовые описания для условия задачи)
- metricType - тип метрики для текущей генерации: 'length' (длина) или 'words' (количество слов)
- current_criterion - полный объект критерия, включающий режим поиска (min/max) и текстовые описания

**Методы генерации данных:**
- generateSentence(size) - генерирует случайное предложение заданного размера с помощью библиотеки Faker и контролирует максимальную длину предложения (maxSentenceSize)
- generateTest() - создает корректные тестовые данные. Генерирует от 2 до N случайных предложений, вычисляет метрику для каждого предложения, определяет строку-ответ согласно критерию (min или max), возвращает кортеж: (входные_данные, ожидаемый_вывод)

**Проверка решения (test):**
- Через регулярные выражения проверяет отсутствие #include <string.h>
- Компилирует код студента с помощью CProgramRunner
- Прогоняет 5 случайных тестов с разными входными данными

**Результат:**
- Если все тесты пройдены → возвращает Result.Ok()
- Если тест не пройден → возвращает Result.Fail(input, expected, got)
- Если возникла ошибка выполнения → возвращает Result.Fail(input, expected, error_message)
